# 리팩토링 구현 계획 - TerminalPanel.cs

## 1. 개요
`TerminalPanel` 클래스는 터미널 관리의 핵심 컨트롤이나, 현재 여러 책임이 섞여 있고 코드가 비대합니다. 이를 구조적으로 정리하여 유지보수하기 쉬운 코드로 개선합니다.

## 2. 주요 수정 사항

### 2.1 터미널 생성 로직 단일화
- `MakeEmulator`, `MakeTabPanel`, `AddTerminal`로 분산된 로직을 통합합니다.
- 터미널 인스턴스 생성 시 공통 이벤트 연결 로직을 한 곳으로 모읍니다.

### 2.2 명령 전송 헬퍼 메서드 도입
- `ShortenScript_OnSendScript`와 `ShortenCommand_OnSendShortenCommand`에서 공통적으로 사용하는 "대상 터미널 결정" 로직을 `ExecuteOnTargetTerminals`와 같은 메서드로 추상화합니다.

### 2.3 비동기 처리 및 예외 처리 강화
- `async void` 사용을 최소화하고(이벤트 핸들러 제외), 내부 로직은 `Task`를 반환하도록 수정합니다.
- `ConnectDeviceAsync` 호출 시 예외 처리를 중앙화하여 사용자에게 일관된 에러 메시지를 제공합니다.

### 2.4 UI 및 유틸리티 정리
- `CheckTerminalCount` 내의 문자열 파싱 로직을 보다 견고하게 개선합니다.
- 사용되지 않는 변수 및 주석 처리된 레거시 코드를 제거합니다.
- 리전(Region) 또는 코드 블록 정리를 통해 가독성을 확보합니다.

## 3. 작업 단계
1. **기반 구조 정리**: 사용하지 않는 필드 제거 및 메서드 접근 제한자 재검토.
2. **명령 전송 로직 리팩토링**: 스크립트/명령 실행부 통합.
3. **터미널 추가/제거 로직 개선**: 생명주기 관리 강화.
4. **최종 검증**: 빌드 확인 및 런타임 동작 확인.
